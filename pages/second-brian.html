Second Brian

Code Project — Active
A living archive of attention. Not a productivity system. Not a static knowledge base.

Second Brian is a long-term personal knowledge system — a Telegram-to-Obsidian AI pipeline that captures thoughts, voice memos, documents, and passing observations and transforms them into structured, permanently linked notes.

Architecture

  • Telegram bot as frictionless capture interface — send anything, anytime
  • AI pipeline (Claude / local LLM via Ollama) for processing and structuring
  • Obsidian vault as the knowledge graph output
  • Rhizomatic note structure — notes link outward to hubs, hubs persist over time
  • Pluggable LLM backend — local or cloud depending on context

Design Principles

  • Notes are events — time-bound snapshots of attention, not timeless truths
  • Contradictions are preserved, never resolved
  • Emotional context and uncertainty are treated as data
  • Markdown-first, tool-agnostic, readable without automation
  • AI-assisted, not AI-directed — meaning stays with the human
